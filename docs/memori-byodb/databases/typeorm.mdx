# TypeORM

Memori auto-detects a TypeORM `DataSource` and routes SQL to the correct dialect. Pass the initialized `DataSource` directly.

## Install

Install TypeORM and your underlying database driver:

<CodeGroup title="Install TypeORM">

```bash {{ title: 'SQLite (better-sqlite3)' }}
npm install typeorm better-sqlite3
npm install --save-dev @types/better-sqlite3
```

```bash {{ title: 'PostgreSQL' }}
npm install typeorm pg
npm install --save-dev @types/pg
```

```bash {{ title: 'MySQL / MariaDB' }}
npm install typeorm mysql2
```

</CodeGroup>

## Quick Start

<CodeGroup title="TypeORM Connection">

```typescript {{ title: 'SQLite' }}
import 'dotenv/config';
import { DataSource } from 'typeorm';
import { OpenAI } from 'openai';
import { Memori } from '@memorilabs/memori';

const dataSource = new DataSource({
  type: 'better-sqlite3',
  database: 'memori-typeorm.db',
});
await dataSource.initialize();

const client = new OpenAI();

const mem = new Memori({ conn: () => dataSource }).llm.register(client);
mem.attribution('user-123', 'my-app');

if (!mem.config.storage) {
  throw new Error('Storage not initialized');
}

await mem.config.storage.build();

const response = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'My favorite color is blue.' }],
});
console.log(`AI: ${response.choices[0]?.message?.content}`);

await mem.augmentation.wait();
await mem.config.storage.close();
await dataSource.destroy();
```

## Supported Dialects

| TypeORM `type`   | Dialect Detected |
| ---------------- | ---------------- |
| `better-sqlite3` | SQLite           |
| `sqlite`         | SQLite           |
| `postgres`       | PostgreSQL       |
| `cockroachdb`    | PostgreSQL       |
| `mysql`          | MySQL            |
| `mariadb`        | MySQL            |

## Notes

- Always call `await dataSource.initialize()` before passing it to `Memori`. TypeORM requires explicit initialization.
- Call `await dataSource.destroy()` after `mem.config.storage.close()` to cleanly shut down the connection pool.
- Memori manages its own raw SQL tables — no TypeORM `Entity` definitions required.

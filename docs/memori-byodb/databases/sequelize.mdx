# Sequelize

Memori auto-detects a Sequelize instance and routes SQL to the correct dialect. Pass the `Sequelize` instance directly.

<Note>
  Sequelize's `mariadb` dialect is not supported. Use MySQL instead — MariaDB is
  wire-compatible with MySQL and works via the `mysql` dialect.
</Note>

## Install

Install Sequelize and your underlying database driver:

<CodeGroup title="Install Sequelize">

```bash {{ title: 'SQLite' }}
npm install sequelize better-sqlite3
npm install --save-dev @types/better-sqlite3
```

```bash {{ title: 'PostgreSQL' }}
npm install sequelize pg pg-hstore
```

```bash {{ title: 'MySQL' }}
npm install sequelize mysql2
```

</CodeGroup>

## Quick Start

<CodeGroup title="Sequelize Connection">

```typescript {{ title: 'SQLite' }}
import 'dotenv/config';
import { Sequelize } from 'sequelize';
import { OpenAI } from 'openai';
import { Memori } from '@memorilabs/memori';

const sequelize = new Sequelize({
  dialect: 'sqlite',
  storage: 'memori.db',
  logging: false,
});

const client = new OpenAI();

const mem = new Memori({ conn: sequelize }).llm.register(client);
mem.attribution('user-123', 'my-app');

if (!mem.config.storage) {
  throw new Error('Storage not initialized');
}

await mem.config.storage.build();

const response = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'My favorite color is blue.' }],
});
console.log(`AI: ${response.choices[0]?.message?.content}`);

await mem.augmentation.wait();
await mem.config.storage.close();
await sequelize.close();
```

## Supported Dialects

| Sequelize `dialect` | Dialect Detected |
| ------------------- | ---------------- |
| `sqlite`            | SQLite           |
| `postgres`          | PostgreSQL       |
| `mysql`             | MySQL            |

## Notes

- Set `logging: false` to suppress Sequelize's default SQL logging, which can be noisy alongside Memori's own output.
- Call `await sequelize.close()` after `mem.config.storage.close()` to release the connection pool.
- Memori manages its own raw SQL tables — no Sequelize model definitions required.
